# Workflow Guidance

Workflow Guidance lets you build and run visual step-by-step process workflows for clinical and administrative tasks. Workflows are flowcharts — sequences of instructions, decision points, and outcomes — that guide staff through structured processes such as handling a discharge summary, processing a fit note, or triaging an incoming task. Each workflow is displayed as an interactive canvas that staff read and navigate in real time.

Where the [Practice Handbook](/features/practice-handbook) stores static policies and reference information, Workflow Guidance handles processes that require decision-making: *if this, then that*.

---

## The staff experience

Staff access Workflow Guidance from the main navigation. The landing page shows available workflows grouped into two sections:

- **Common documents (daily)** — the Primary workflows used regularly by most staff
- **Less common / specialist** — Supporting workflows for less frequent tasks, or for handling specific parts of a larger process

An **Admin** section at the bottom of the landing page provides links to Manage Templates, Engagement, and Node styling defaults (visible to admins only).

To use a workflow, staff click **Start workflow** on the relevant card. This opens the workflow canvas in Viewing mode. The canvas is the staff experience — there is no separate step-by-step modal. Staff read the flowchart, follow the decision paths, and tap the **ℹ button** on any node to open its detailed body text in the Details panel on the right. The Details panel also contains an expandable **Diagram legend** explaining the node types.

When starting a workflow, staff can optionally add:

- A **reference** (e.g. "Cardiology letter — JS 12 Jan") — this helps locate the instance later in the audit trail
- A **category** (e.g. "Hospital Letter") — this filters engagement analytics

---

## Canvas modes

The workflow canvas has two modes, toggled from the Details panel:

**Viewing mode** — Read-only. Shows the full flowchart with all nodes, connections, and outcome labels. Staff work in Viewing mode. The Details panel shows the Diagram legend and, when a node is selected, that node's reference details and body text.

**Editing mode** — Full edit mode for admins building or modifying a workflow. The canvas shows an orange border and an "Editing mode active" indicator. The toolbar above the canvas becomes active, showing buttons to add each node type. Drag nodes to reposition them; hold Shift to lock to an axis.

## Node types

Workflows are built from five node types. Each serves a distinct purpose.

| Node type | Canvas shape | When to use |
|---|---|---|
| Instruction | Rectangle | Any step where staff need to read or check something before proceeding |
| Question | Diamond | When the next action depends on the answer to something — signals a decision point |
| Outcome | Rounded rectangle (shown as "END") | When a workflow path is complete — always end each path with an Outcome |
| Panel | Background container | To group related steps visually; does not affect workflow logic |
| Reference | Sidebar card | Supporting information staff might need, shown alongside the flow |

### Instruction

Use for steps where staff need to read, check, or act on something before continuing. Instruction nodes can contain a detailed body (visible when staff tap ℹ) including checklists, guidance text, or diagnosis codes.

*Example:* "Code Diagnoses — check the discharge summary diagnosis matches one of the listed codes and code if appropriate."

### Question

Use when the next action depends on the answer to something. The diamond shape signals a decision point with branching paths.

*Example:* "GP Actions?" with Yes → a further instruction node, No → proceed to outcome.

### Outcome

Use when a workflow path is complete. Always end each path with an Outcome node so staff know what to do with the document or task. Outcome nodes display a title and an "Outcome:" sub-line summarising the action. On the canvas, Outcome nodes are labelled **END**.

### Panel

Use to group related steps visually, especially in complex workflows where it helps to separate phases (e.g. "Initial Triage", "Clinical Review", "Action"). Panels do not affect workflow logic — they are purely visual organisers.

### Reference

Use for supporting information that staff might need but that isn't part of the decision flow — for example, a reference card listing local referral criteria, shown alongside the workflow.

### Node options

All node types support:

- **Node styling** — custom background and text colours, configurable per-node
- **STAMP badge** — a short badge label displayed on the node in the canvas view

Instruction nodes additionally support **linked workflows** — cross-references to other workflow templates that are accessible from the Details panel when that node is selected.

## Workflow types and landing sections

Each workflow template has a **type** that controls where it appears on the staff landing page:

- **Primary** — Shown in "Common documents (daily)". These are the workflows most staff use regularly.
- **Supporting** — Shown in "Less common / specialist". These are used less frequently or are designed to be linked from other workflows to handle a specific sub-process.

Templates also carry a **source** badge:

- **Global** — Provided centrally; not customised by the surgery
- **Customised** — A surgery-modified version of a global template (shown on both the landing page card and the template list)
- No badge — Created entirely by the surgery

## Creating and editing workflows

Admins manage workflow templates from **Manage templates** (accessible from the landing page or the Admin section).

To create a new workflow: select **New template**, choose a type (Primary or Supporting), and add a title. The canvas opens in Editing mode ready to build.

To edit an existing workflow: open the template and switch to Editing mode from the Details panel. The toolbar above the canvas shows:

- **Add instruction** — adds an Instruction node
- **Add question** — adds a Question node
- **Add outcome** — adds an Outcome node
- **Add panel** — adds a Panel container
- **Add reference** — adds a Reference card

Drag nodes to position them on the canvas. Connect nodes by drawing from one node's output handle to another's input handle. Hold Shift while dragging to lock movement to a horizontal or vertical axis.

When a node is selected in Editing mode, the Details panel on the right shows the node editor where you can set the node's title, body text, styling, STAMP badge, and (for Instruction nodes) any linked workflows.

## Draft and approval lifecycle

Workflow templates move through a defined lifecycle:

**Draft** — A template being built or revised. Not visible to staff. Admins can edit a Draft template freely.

**Approved** — A template that has been reviewed and published. Visible to staff on the landing page.

**Superseded** — A template that was previously Approved but has been replaced by a newer version. The previous version is retained for audit purposes and is visible in the template history, but is no longer shown to staff as an active workflow.

To publish a Draft template, an admin with appropriate permissions marks it as Approved. Any subsequent edits create a new Draft version; approving that new Draft supersedes the previous Approved version.

## Engagement analytics

The Engagement section (accessible from the Admin area on the landing page) shows how often each workflow is being used and by which staff members. Results are filterable by time period.

If certain workflows show low engagement, it may indicate they aren't being found — consider adding a link from the Practice Handbook, or surfacing them via a Quick Access button. If a workflow is used very frequently, it may be a candidate for review and optimisation.
